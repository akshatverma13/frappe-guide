The Biggest Shift

In mForm, conditions are distributed across parent/child arrays and regex patterns on multiple questions. In Frappe, each field carries its own self-contained "depends on" expression. One line, on the field itself. Much easier to read and maintain.

Skip Logic — Side by Side

Conditional visibility in mForm was distributed across multiple questions:

  • Parent/child arrays referencing question IDs
  • Restriction type 11 for DID relations
  • Regex like parent_option: ^([1])$
  • AND logic by default, OR via special VAL_OR_CASE flag
⚠️
Problem

To understand when one field appeared, you traced arrays and regex across many questions.

In Frappe, each field carries its own condition:

doc.purpose_of_visit == "Training"

→ This Training section only appears when Purpose = Training. One line. On the field itself.

AND logic: doc.a == "X" && doc.b == "Y"

OR logic: doc.a == "X" || doc.a == "Y"

✅
Result

To understand when a field appears, read one expression on that field. No tracing.

mFormFrappe
parent/child arrays + type 11"Depends On" expression on the field
