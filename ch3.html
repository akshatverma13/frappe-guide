The Biggest Shift

In mForm, conditions are distributed across parent/child arrays and regex patterns. In Frappe, each field carries one self-contained "depends on" expression. One line, on the field itself. Much easier to read and maintain.

Skip Logic — Side by Side

Conditional visibility in mForm was distributed across multiple questions:

  • Parent/child arrays referencing question IDs
  • Restriction type 11 for DID relations
  • Regex patterns like parent_option: ^([1])$
  • AND logic by default, OR via special VAL_OR_CASE flag
⚠️
Problem

To understand when one field appeared, you traced arrays and regex patterns across many questions.

In Frappe, each field carries its own self-contained condition:

doc.purpose_of_visit == "Training"

→ Training section appears only when Purpose = Training. One expression, on the field itself.

AND: doc.a == "X" && doc.b == "Y"  ·  OR: doc.a == "X" || doc.a == "Y"

✅
Result

To understand when a field appears, read one expression on that field. No tracing required.
