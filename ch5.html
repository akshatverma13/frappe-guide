🔑
The Superpower — True Offline

Fill forms and save with zero internet. Data stores locally on the device and syncs to the server whenever connectivity returns. No data loss. No time limit. Indefinitely.

App Architecture — Thin Shell, Fat SDK

🏠
App Shell

Lightweight. Provides branding, project config, and client-specific Dart form handlers. Different per client project.

⚙️
SDK Engine

The heavy lifter: reads DocType definitions, renders forms as native screens, stores data locally (SQLite), syncs with server, manages auth tokens. Shared across all projects.

💡
No form is hardcoded

Every form is built dynamically at runtime from the server's DocType definition. Form changes on the server update the app automatically on next sync.

Login — What Happens

1

App checks if mobile service is enabled

Quick server check, no credentials needed. If disabled, stops here.

2

User enters credentials

Username/password or mobile + OTP. On success: receives auth tokens (30-day validity), enabled forms list, permissions, language preference.

3

Initial sync

Downloads all form definitions, existing records, and translations.

4

App is ready — internet can be turned off

Everything for offline work is now stored locally. The user is fully self-sufficient.

Record Status

StatusMeaningWhat Happens
CleanIn sync with server. No local changes.Nothing — it is up to date.
DirtyHas local changes not yet uploaded.Uploads on next sync.
ErrorServer rejected it during sync.User sees error, fixes, re-saves.
DeletedMarked for deletion locally.Delete sent to server on next sync.

Online vs Offline Save

  1. Data sent to server immediately
  2. Server runs all Python validation scripts
  3. If passes → server assigns permanent ID, record becomes Clean
  4. If fails → error shown to user, record not saved anywhere
  1. Data saved to local SQLite database on the phone
  2. App assigns a temporary local ID
  3. Record marked Dirty — needs to upload
  4. Only mobile-side validation runs (server checks pending)
  5. Auto-uploads when internet returns
✅
User experience

The Save button works identically online or offline. Users never need to check connectivity.

  1. Record stays Dirty with error details stored locally
  2. User sees which record failed and exactly why
  3. User opens record, fixes the issue, re-saves
  4. Auto-retries on next sync
  5. Other records are not blocked — each syncs independently
