The Three Script Layers

Frappe splits business logic across three layers. Understanding which layer to use for which problem is essential:

🐍
Server Script (Python)
Runs on the server on every save/submit/cancel. The safety net — always executes regardless of how data arrives.
🌐
Web Client Script (JavaScript)
Runs in the browser for Frappe Desk users only. Real-time interactive behavior — calculations, filters, custom UI.
📱
Mobile Form Handler (Dart)
Runs in the Flutter app. Same real-time behavior for mobile — on_field_change, on_validate. App-only.

Server Scripts — Python

Server Scripts run on the Frappe server on every document event — before_save, after_save, before_submit, on_submit, etc. They are the authoritative validation layer.

# frappe_app/custom/validation.py import frappe def validate(doc, method): # Cross-field validation if doc.end_date < doc.start_date: frappe.throw("End date must be after start date") # Numeric range check if doc.attendance_count > doc.total_learners: frappe.throw("Attendance cannot exceed total learners") # Conditional mandatory (not enforced by form definition) if doc.cms_status == "Running" and not doc.batch_number: frappe.throw("Batch Number is required when CMS Status is Running") # Auto-calculation before save doc.total_score = doc.reading_score + doc.writing_score + doc.math_score

Web Client Scripts — JavaScript

Client Scripts run in the browser only — for Frappe Desk users. They provide real-time interactive behavior that goes beyond what the form definition can express.

// frappe.ui.form.on — triggered when form loads or field changes frappe.ui.form.on("Daily Visit Report", { // Called when form is refreshed/loaded refresh: function(frm) { if (frm.doc.status === "Approved") { frm.disable_form(); // lock all fields } }, // Triggered when a specific field changes reading_score: function(frm) { // Recalculate total when any score changes frm.set_value("total_score", (frm.doc.reading_score || 0) + (frm.doc.writing_score || 0) + (frm.doc.math_score || 0) ); }, // Custom filter on a Link field volunteer_id: function(frm) { frm.set_query("volunteer_id", function() { return { filters: { village: frm.doc.village } }; }); } });

Frappe Hooks — Event-driven Integration

Hooks (defined in hooks.py of a Frappe App) let you attach Python functions to Frappe events without modifying core code:

doc_events
Run functions before/after any Document event: before_save, after_insert, on_submit, on_cancel, on_trash
scheduler_events
Schedule functions to run daily, weekly, hourly: daily, weekly, monthly, cron
override_whitelisted_methods
Replace or extend any Frappe API endpoint with your own implementation
on_session_creation
Run code when a user logs in — useful for loading user-specific configuration
fixtures
Define data (DocTypes, roles, custom fields) that should be auto-installed with the app
# hooks.py — attach validation function to DVR DocType doc_events = { "Daily Visit Report": { "validate": "myapp.validation.dvr.validate", "on_submit": "myapp.notifications.dvr.send_approval_alert", }, "*": { # runs for ALL DocTypes "after_insert": "myapp.audit.log_creation", } }

REST API — Built-in for Every DocType

Frappe auto-generates REST API endpoints for every DocType. No extra code required:

OperationHTTP MethodEndpoint
List recordsGET/api/resource/DocType?filters=[[...]]&fields=[...]
Get one recordGET/api/resource/DocType/record-name
Create recordPOST/api/resource/DocType
Update recordPUT/api/resource/DocType/record-name
Delete recordDELETE/api/resource/DocType/record-name
Custom methodGET/POST/api/method/app.module.function

Custom Frappe Apps

A Frappe App is a Python package that extends Frappe with new DocTypes, scripts, configurations, and fixtures. All customizations in production should live in a custom app — not in Frappe core.

# Create a new Frappe app bench new-app myapp # App directory structure myapp/ myapp/ __init__.py hooks.py # Event hooks and configuration modules.txt # List of modules public/ # Static files (JS, CSS) templates/ # Jinja templates module_name/ doctype/ my_doctype/ # DocType definition files report/ # Custom reports page/ # Custom pages # Install on a site bench --site mysite.local install-app myapp
