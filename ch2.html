Field Types — Complete Reference

Dropdown with fixed choices hardcoded in the field. Use for small stable lists (e.g., Gender).

Dropdown that pulls from another DocType. Use for large/growing lists (e.g., State, Volunteer).

💡
Select vs Link

Use Select for short lists that never change (Gender: Male/Female/Other). Use Link when the list might grow or is shared across forms — adding new options requires no form editing.

Section Break
Starts a new visual card/block. Can have a title. Can be collapsible.
Tab Break
Creates a new tab at the top of the form — like pages in a long form.
Column Break
Arranges next fields side-by-side instead of stacking.
HTML
Displays formatted text or embedded content on the form.
Attach
File upload — any file type
Attach Image
Photo upload. On mobile, opens camera directly.
Rating
Star rating (1–5)
Read Only
Displays a value the user cannot edit.
Button
Clickable button that triggers a server action.
Duration
Time duration in hours/minutes.

Child Tables — Forms Inside Forms

When a form needs a repeating list, use a Child Table. It appears as an "Add Row" section.

📊
Example: Household Survey → Family Members
#NameAgeGenderEducationReading
1Ramesh Kumar42MalePrimaryYes
2Sunita Kumar38FemaleIlliterateNo
3Priya Kumar14FemaleSecondaryYes

Each row saves with the parent form. Rows can be added, edited, reordered, or deleted.

How Fields Connect

🔗Cascading Dropdowns (Link Filters)▼

Selecting a State narrows District. Selecting a District narrows Block. Works for any dependent pair, not just geography.

State: Maharashtra → District: Pune ▼ → Block: Haveli ▼

When State changes → District resets. When District changes → Block resets.

✨Auto-fill (Fetch From)▼

Select a Volunteer → their name, designation, and GPS auto-populate from the Volunteer record. Auto-filled fields are read-only. Works on web and mobile from the same definition.

👁️Conditional Visibility (Depends On)▼

Each field has a self-contained "depends on" expression. The field appears only when the condition is true.

doc.purpose_of_visit == "Training"

→ The Training section only appears when Purpose of Visit equals "Training".

⚠️
When do you need a developer?

No dev: add field, change label, add option, set default. Dev needed: new validation, auto-calculation between fields, custom dropdown filtering.
