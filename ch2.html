Select vs Link

Use Select for small stable lists (Gender). Use Link when the list might grow or is shared across forms — new options added without touching any form definition.

Child Tables — Forms Inside Forms

When a form needs a repeating list, use a Child Table. It appears as an "Add Row" section where users add multiple entries.

📊
Example: Household Survey → Family Members
#NameAgeGenderEducationReading
1Ramesh Kumar42MalePrimaryYes
2Sunita Kumar38FemaleIlliterateNo
3Priya Kumar14FemaleSecondaryYes

Each row saves with the parent form. Rows can be added, edited, reordered, or deleted.

How Fields Connect

🔗Cascading Dropdowns▼

Selecting a State narrows the District list. Selecting District narrows Block. Applies to any dependent pair.

Maharashtra → Pune ▾ → Haveli ▾
✨Auto-fill (Fetch From)▼

Select a Volunteer → their name, designation, and GPS auto-populate. Auto-filled fields are read-only. Defined once, works on web and mobile.

👁️Conditional Visibility (Depends On)▼

Each field has a self-contained "depends on" expression. The field shows only when the condition is true.

doc.purpose_of_visit == "Training"

→ Training section only appears when Purpose = Training.

⚠️
When do you need a developer?

No dev needed: Add field, change label, add dropdown option, set default value.
Dev required: New validation rule, auto-calculation, custom dropdown filtering.
