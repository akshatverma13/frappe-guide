Anatomy of a DocType

A DocType is the blueprint for a data collection form. It contains everything Frappe needs to build the form, the database table, and the API:

📋
Fields

Types, labels, required status, default values, description, and field-level options.

📐
Layout

Section Breaks, Tab Breaks, and Column Breaks control visual grouping and arrangement.

🔒
Permissions

Which roles can Read, Create, Write, Delete, Submit, Cancel, or Amend records.

🏷️
Naming Rule

How each record gets its unique ID — auto-increment, user-defined, or formula-based.

⚙️
Settings

Is Submittable? Is a Child Table? Has Timeline? Editable after Submit? Track Changes?

📎
Links & Actions

Links to other DocTypes shown in the sidebar. Custom buttons and quick-action menu items.

All Field Types

Data
Single line of text. Names, IDs, short answers. Max 140 chars by default.
Small Text
Multi-line text, shorter entries. Rendered as textarea.
Long Text
Multi-line text for longer content. No character limit.
Text Editor
Rich text (WYSIWYG) editor with formatting support.
Markdown
Markdown-formatted text with preview rendering.
Int
Whole numbers. Counts, quantities, scores.
Float
Decimal numbers. Percentages, measurements.
Currency
Monetary values with locale-aware formatting.
Percent
Percentage value, stored as a float (0–100).
Check
A single yes/no boolean checkbox. Stored as 0 or 1.
Phone
Phone number with international format support.
Password
Masked text entry. Stored encrypted in database.
Select
Dropdown with fixed choices hardcoded in the field definition. Best for small, stable lists.
Link
Dropdown that pulls records from another DocType. Has autocomplete search. Enforces referential integrity.
Dynamic Link
Link to any DocType — the target DocType is determined by another field's value.
Table
Embeds a child table — a repeating "Add Row" grid inside the parent form.
Table MultiSelect
Multi-select that stores multiple linked values from another DocType.
Autocomplete
Free-text input with suggestions fetched from a Link field's options.
💡
Select vs Link

Select for short, stable lists hardcoded in the field (e.g., Gender). Link for lists that may grow or are shared across forms — new options are added as records in the linked DocType, no form editing required.

Date

Datetime
Date + time picker. Stored as YYYY-MM-DD HH:MM:SS.
Time
Time-only picker. Stored as HH:MM:SS.
Duration
Time duration (hours, minutes, seconds) with a visual picker.
Attach
File upload. Any file type. Stored on server, URL saved in the field.
Attach Image
Image upload. On mobile, can open camera directly. Preview shown inline.
Geolocation
GPS coordinates stored as JSON with lat/lng/altitude. On mobile, captures device location.

Star rating widget (1–5 stars). Stored as integer.

Starts a new visual card/block. Can have a label. Can be collapsible. Groups fields visually.

Splits the section into side-by-side columns. The next fields appear in a new column.

Creates a new tab at the top of the form — like pages within a long form.
HTML
Displays formatted HTML content, images, or embedded elements on the form.
Fold
Everything after this field is hidden until the user clicks "Show more".
Heading
A non-input heading text displayed inside the form for visual organization.

Displays a computed or fetched value. User cannot edit. Used for auto-calculated fields.
Button
A clickable button that triggers a client-side or server-side action.
Barcode
Barcode/QR code scanner. On mobile, opens the device camera.

Child Tables — Repeating Data Inside Forms

When a form needs a list of items, use a Child Table. It appears as an "Add Row" section where users add multiple entries, each with their own fields.

📊
Example: Household Survey → Family Members
#NameAgeGenderEducationReadingWriting
1Ramesh Kumar42MalePrimaryYesYes
2Sunita Kumar38FemaleIlliterateNoNo
3Priya14FemaleSecondaryYesYes

Each row is one record in the Family Member child DocType. All rows save with the parent form. Rows can be added, edited, reordered, and deleted.

✅
Key Rule: Child Tables cannot exist alone

A Child Table DocType is marked "Is Child Table" — it can only exist inside a parent form's Table field. It never has its own List View.

Naming Rules — How Records Get Their IDs

Naming TypeExampleWhen to Use
Auto-increment SeriesHH-00001, SPDVR-00042Transaction data — needs a unique, readable ID
Set by UserMaharashtra, Pune, EnglishMaster data where the name IS the identifier
ExpressionEMP-2024-0001IDs that include date, category, or other fields
UUIDab3f8c12-...When human-readable IDs are not needed
Based on FieldUses the value of a specific fieldWhen one field uniquely identifies the record

Master Data vs Transaction Data

📚 Master Data
What it isReference data, created once and rarely changed
ExamplesState, District, Employee, Volunteer, Education Level
Created byAdministrators during setup
Used byTransaction forms that Link to it
Must exist firstYes — referenced by transaction forms
⚡ Transaction Data
What it isField-collected data, created frequently
ExamplesDaily Visit Report, Attendance, Household Survey
Created byField workers on the mobile app
Used byReports, dashboards, approvals
Must exist firstNo — references master data
