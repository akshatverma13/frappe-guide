Glossary

Every term used across the guide, defined in plain English. Bookmark this page.

Core Platform Terms

DocType
A form template — the blank structure that defines what fields exist, their types, layout, and rules. Not the data itself.
Document
A filled-in form — one submission. Every saved entry creates one Document.
Module
A folder grouping related DocTypes. e.g., "Shiksha Plus" module contains DVR, MIS, Attendance forms.
Child Table
A repeating "Add Row" section inside a form. Each row is a mini-form saved with the parent.
Master Data
Reference data created once and reused by many forms: geography, employees, option lists.
Transaction Data
Field-collected data: visit reports, attendance, assessments. References master data.
Frappe Desk
The browser-based back office. No installation. Used by admins and coordinators.
Option Master
A small DocType storing dropdown choices (e.g., Education Level). Add options without touching any form.
Fixtures
Pre-loaded reference data (all 30+ option masters) automatically installed with the system.
Mobile Configuration
Admin screen controlling which forms appear on the mobile app and in what categories.

Field & Form Terms

Depends On
A condition expression on a field that controls when it shows or hides. Self-contained on each field.
Link Field
A dropdown that pulls choices from another DocType. e.g., State dropdown pulling from the State master.
Fetch From
An auto-fill rule on a field. When a linked record is selected, copies a value from that record.
Server Script
Python code that validates/processes data on the server every time a form is saved. The safety net.
Mobile Handler
Dart code in the app. Handles real-time field calculations and on-validate checks on mobile.
Client Script
JavaScript code on Frappe Desk. Provides real-time behaviour on the web interface only.

Sync Terms

Schema Sync
App downloads updated form definitions from server. Triggered by form changes. Automatic.
Push Sync
App uploads filled records to server. Follows dependency ordering (geography first).
Pull Sync
App downloads records from server — other users' data, Desk edits, new master data.
Delta Sync
Only records modified since the last sync are downloaded. Not the entire database every time.
Dirty Record
A locally saved record with unsynchronised changes. Uploads on next sync.
Clean Record
A record fully in sync with the server. No pending changes.
