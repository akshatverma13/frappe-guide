Glossary

Every term used across the guide, defined in plain English. A quick reference for the entire team.

Core Platform Terms

DocType
A form template — the blank structure defining what fields exist, their types, layout, and rules. Not the data itself.
Document
A filled-in form — one submission. Every saved entry creates one Document with a unique ID.
Module
A folder grouping related DocTypes. e.g., "Shiksha Plus" contains DVR, MIS, Attendance forms.
Child Table
A repeating "Add Row" section inside a form. Each row is a mini-form saved with the parent.
Master Data
Reference data created once and reused by many forms: geography, employees, option lists.
Transaction Data
Field-collected data: visit reports, attendance, assessments. References master data.
Frappe Desk
The browser-based back office. Used by admins and coordinators. No installation required.
Option Master
A small DocType storing dropdown choices (e.g., Education Level). Add options without touching any form.
Fixtures
Pre-loaded reference data — all 30+ option masters installed automatically with the system.
Mobile Configuration
Admin screen controlling which forms appear on the mobile app and in what category groupings.

Field & Form Terms

Depends On
A condition expression on a field controlling when it shows or hides. Self-contained on each field.
Link Field
A dropdown pulling choices from another DocType. e.g., State dropdown from the State master.
Fetch From
An auto-fill rule — when a linked record is selected, copies a value from that record into this field.
Server Script
Python code validating data on the server every time a form is saved. The safety net for all rules.
Mobile Handler
Dart code in the app for real-time field calculations and on-validate checks on the mobile side.
Client Script
JavaScript on Frappe Desk providing real-time behaviour on the web interface only.

Sync Terms

Schema Sync
App downloads updated form definitions from the server. Triggered by any form change. Automatic.
Push Sync
App uploads filled records to the server. Follows dependency ordering (geography first).
Pull Sync
App downloads records from server — other users' data, Desk edits, new master data.
Delta Sync
Only records modified since the last sync are downloaded. Not the full database every time.
Dirty Record
A locally saved record with unsynchronised changes. Uploads automatically on next sync.
Clean Record
A record fully in sync with the server. No pending local changes.
