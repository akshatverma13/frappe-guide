Full Glossary

Every term used throughout this guide. A complete reference for the entire team.

Core Framework Terms

Frappe
An open-source, meta-driven web application framework built on Python. Generates database tables, web forms, and REST APIs from DocType definitions.
DocType
A form template / data model. Defines fields, layout, permissions, naming rules, and behavior. The central building block of Frappe.
Document
One filled-in record — a single instance of a DocType. "Household Survey HH-00001" is a Document.
Module
A logical grouping of related DocTypes. Appears as a folder in the Frappe Desk sidebar.
Frappe App
A Python package that extends Frappe with new DocTypes, scripts, and configurations. All customizations should live in a Frappe App.
Frappe Desk
The browser-based back office UI that comes with every Frappe installation. No installation required — works in any browser.
Bench
The CLI tool and directory structure for managing Frappe installations. Used to create sites, install apps, run migrations, and update.
Site
A single Frappe deployment — one URL, one MariaDB database, one set of users and data. Managed by Bench.
Fixtures
Pre-loaded data bundled with a Frappe App. Auto-installed on every fresh deployment — ensures consistent base data.
Hooks
Event bindings in hooks.py that attach Python functions to Frappe events (before_save, on_submit, etc.) without modifying core code.

Form & Field Terms

depends_on
A condition expression on a field controlling when it shows or hides. Self-contained on each field. Same on web and mobile.
Link Field
A dropdown that pulls choices from another DocType. Enforces referential integrity — the linked record must exist.
fetch_from
An auto-fill property. When a linked record is selected, copies a specified value from that record into the current field.
Link Filter
A filter applied to a Link field's dropdown based on another field's value. Used for cascading dropdowns (State → District).
Child Table
A DocType marked as a child — a repeating list inside a parent form. Cannot exist independently, has no List View.
Option Master
A small DocType that stores dropdown choices. Link fields point to it. Adding new options requires no form editing.
mandatory_depends_on
Makes a field required only when a condition is met. Must also be enforced by a Server Script on the server side.
Permlevel
A 0–9 number on each field. Roles need the matching permission level to see or edit that field. Used for field-level access control.

Mobile & Sync Terms

Mobile SDK
The reusable Flutter/Dart engine that handles form rendering, offline storage, sync, and auth. Shared across all mobile projects.
Dart Form Handler
A Dart class providing on_field_change and on_validate behavior on mobile. The mobile equivalent of a Frappe Client Script.
Mobile Controller
The pattern of using Dart Form Handlers to attach project-specific logic to the SDK's form rendering engine.
Schema Sync
Downloads updated DocType definitions from server to app. Triggered by form changes. No app store update needed.
Push Sync
Uploads Dirty records (new, modified, deleted) from app to server. Follows dependency ordering.
Pull Sync
Downloads records from server to app — other users' data, Desk edits, new master data. Uses delta approach.
Delta Sync
Only downloading records modified since the last sync timestamp. Efficient even with large datasets.
Dirty Record
A locally saved record with unsynchronised changes. Automatically uploaded on next sync cycle.
Clean Record
A record fully synchronized with the server. No pending local changes.
Offline-First
Design principle: the app works fully without internet. Data saved locally, synced when connected. No data loss.

Script & Permission Terms

Server Script
Python code that runs on the server on every document event. The authoritative validation and calculation layer.
Client Script
JavaScript code that runs in the browser (Frappe Desk only). Provides real-time interactive behavior on the web.
Role
A named permission bundle (e.g., "Field Coordinator"). Users are assigned roles. Roles define what they can do with each DocType.
User Permission
Restricts a user to only see/edit records where a specific link field matches their identity. Implements "own records only".
Workflow
A state machine for a DocType — defines states (Draft/Submitted/Approved) and which roles can trigger transitions between them.
Amendment
Creating a new editable version of an approved/submitted record, while preserving the original for audit trail purposes.
