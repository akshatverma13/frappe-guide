What Is Frappe?

Frappe is an open-source, low-code web application framework built on Python and JavaScript. Its core idea is meta-driven development — instead of writing code to create every form, table, and page, you define a DocType (a metadata description), and Frappe automatically generates the database table, the web form, the API endpoints, and the mobile representation.

🔑
The Core Idea

In most frameworks, adding a new form means writing models, controllers, views, and migrations. In Frappe, you create a DocType definition and everything is generated for you — database, web UI, REST API, and mobile form.

Why Frappe? — Key Advantages

⚡
Rapid Development

Adding a new form takes minutes, not days. Define the DocType, and Frappe generates the database table, web form, API, and mobile form automatically.

🔄
One Definition, Two Interfaces

Define a form once — it works on both the web browser and the Android mobile app. Change it once, both update automatically. No duplicate work.

📶
Offline-First Mobile

The mobile app stores everything locally and syncs when internet is available. Field workers can collect data in areas with no connectivity.

🔓
Open Source & Self-Hosted

Frappe is fully open source (MIT license). You control your data, your server, and your customizations. No vendor lock-in.

Meta-driven Development — How It Works

📐Step 1 — Define a DocType (metadata)▼

You create a DocType definition — essentially a configuration file describing fields, types, required status, layout, and permissions. This is done visually in Frappe Desk, no coding required for standard forms.

🗄️Step 2 — Frappe creates the database table▼

When you save a DocType, Frappe automatically creates or alters the corresponding MariaDB table. Add a field — the column is added. Remove a field — the column is archived. No SQL migrations needed.

🌐Step 3 — Web form and API are generated▼

Frappe Desk automatically generates a List View (table of all records) and a Form View (the full form) for every DocType. REST API endpoints are also generated: GET /api/resource/DocType, POST /api/resource/DocType, etc.

📱Step 4 — Mobile app reads the same definition▼

The mobile SDK downloads the DocType definition from the server and dynamically renders it as native Android screens. The same fields, conditions, and options — no separate mobile form configuration needed.

Key Terminology

DocType
A form template / data model definition. Defines fields, layout, rules, and permissions.
Document
A single record / instance of a DocType. One filled-in form = one Document.
Module
A logical grouping of related DocTypes. Like a folder in the sidebar.
Frappe Desk
The browser-based back office UI that comes with every Frappe installation.
Frappe App
A Python package that extends Frappe with new DocTypes, scripts, and features.
Site
A single Frappe deployment / database — one URL, one set of users and data.
Bench
The command-line tool and directory structure that manages Frappe installations.
Fixtures
Pre-loaded data (records) that are bundled with a Frappe app and installed automatically.
