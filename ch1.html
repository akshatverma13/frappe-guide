System Overview

What the platform is, how its three layers connect, and the six terms you must understand before everything else.

DocTypeDocumentModuleChild Table3 Layers

The Core Principle

Define a data collection form once on the server — it automatically works on both the web browser and the Android mobile app. Change it on the server and both update automatically. No app store update ever needed.

🔑
One definition. Two interfaces. Automatic sync.

This is the founding principle of the entire platform.

The Three Layers

🧠
Server — Frappe
The brain. Stores form definitions, collected data, user accounts, permissions, and all business rules.
🖥️
Frappe Desk — Web Interface
Browser-based back office. Coordinators and admins review data, configure forms, manage users. No installation needed.
📱
Mobile App — Android
Field workers collect data. Works completely offline. Syncs to server when internet returns. No data loss.

Six Terms to Know

📄DocType — The Form Template▼

A DocType is a form blueprint — defines fields, types, required status, layout, permissions, and naming rules. It is the blank form structure, not the data itself.

✅
mForm equivalent

In mForm, this was a Form JSON file. In Frappe, it's a DocType stored and editable on the server.

📝Document — A Filled-in Form Submission▼

Every time someone fills a DocType and saves, that creates one Document. 500 households surveyed = 500 Documents of the Household Survey DocType, each with a unique ID like HH-00001.

📁Module — A Folder Grouping Related DocTypes▼

A Module is a category. All Shiksha Plus forms live in the "Shiksha Plus" module. All geography forms live in "Geo Master." In Frappe Desk, you navigate by module from the left sidebar.

📊Child Table — A Repeating List Inside a Form▼

A Child Table is an "Add Row" section inside a form for repeating entries. Example: Household Survey has a Family Members child table — each row has name, age, gender, education.

✅
mForm equivalent

Replaces Looping Questions (types 20/21) in mForm.

📚Master Data — Reference Data▼

Master Data is created once and referenced by many forms: states, districts, employees, volunteers, dropdown option lists. Must be created before transaction forms that reference it.

📊Transaction Data — Field-Collected Data▼

Transaction Data is the actual field work: visit reports, attendance, assessments, surveys. These reference master data — a visit report links to a volunteer and a geography location.

How Everything Connects

1

Admin defines a DocType on the server

Creates the form template in Frappe Desk — adds fields, conditions, and permissions.

2

Form appears instantly on Frappe Desk

Any authorized web user can fill it in the browser immediately.

3

Admin enables the form for mobile

In Mobile Configuration, enables the form. App picks it up on next sync.

4

Field workers fill the form — even offline

Data saves locally on the device. Syncs to server when internet returns.

5

Coordinators review and approve on Desk

See all submitted data, approve or reject, add comments, run reports.

Smart Features

🔗
Cascading Dropdowns

Select State → District list narrows. Select District → Block narrows. Works on web and mobile from the same definition.

👁️
Conditional Fields

Fields show or hide based on user input. Defined once — works identically on web and mobile.

✨
Auto-fill from Links

Select a Volunteer → their name, designation, GPS auto-populate. Reduces data entry errors.

🛡️
Layered Validation

Simple rules in form definition. Complex rules in Server Scripts. Real-time feedback via Client Scripts.
