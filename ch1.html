System Overview

What the platform is, how its three layers connect, and the six terms you must know before everything else.

DocTypeDocumentModule3 LayersOffline

The Core Idea

Define a data collection form once on the server — it automatically works on both a web browser and the Android mobile app. Change the form on the server and both update automatically. No app store update ever needed.

🔑
One definition. Two interfaces. Automatic sync.

This is the founding principle of the entire platform.

The Three Layers

🧠
Server — Frappe
The brain. Stores form definitions, collected data, user accounts, permissions, and all business rules.
🖥️
Frappe Desk — Web Interface
Browser-based back office. Coordinators and admins review data, configure forms, manage users. No installation.
📱
Mobile App — Android
Field workers collect data. Works completely offline. Syncs to server when internet returns. Nothing lost.

Six Terms to Know

Learn these six terms and everything else will click into place.

📄DocType — The Form Template▼

A DocType is a form blueprint. It defines what fields exist, their types, which are required, the layout, who can fill it, and how records are named. Think of it as the blank form — the structure, not the data.

Example: "Household Survey" is a DocType. It defines the form structure.

✅
mForm equivalent

In mForm this was a Form JSON file. In Frappe it's a DocType, stored and editable on the server.

📝Document — A Filled-in Form▼

Every time someone fills a DocType and saves, that creates one Document. 500 households surveyed = 500 Documents of the Household Survey DocType. Documents get unique IDs like HH-00001, HH-00002.

📁Module — A Folder of DocTypes▼

A Module is a category grouping related DocTypes. All Shiksha Plus forms live in the "Shiksha Plus" module. All geography forms live in "Geo Master." In the Frappe Desk sidebar you navigate by module.

📊Child Table — A Repeating List Inside a Form▼

A Child Table is an "Add Row" section inside a form for repeating entries. Example: Household Survey has a Family Members child table — each member is one row with name, age, gender, education.

✅
mForm equivalent

This replaces Looping Questions (types 20/21) in mForm.

📚Master Data — Reference Data▼

Master Data is created once and referenced by many forms: states, districts, employees, volunteers, dropdown option lists. Must be created before transaction forms that reference it.

📊Transaction Data — Field-Collected Data▼

Transaction Data is the actual work: visit reports, attendance, assessments, surveys. These reference master data — a visit report references a volunteer, a geography, and dropdown options.

How It All Connects

1

Admin defines a DocType on the server

Creates the form template in Frappe Desk — adds fields, sets conditions, defines permissions.

2

Form appears instantly on Frappe Desk

Any authorized web user can fill it in the browser immediately.

3

Admin enables the form for mobile

In Mobile Configuration, checks the form on. The app picks it up on next sync.

4

Field workers fill the form — even offline

Data saves locally. Syncs to server when internet returns.

5

Coordinators review and approve on Desk

See all submitted data, approve or reject, add comments, export reports.

Smart Features That Work Everywhere

🔗
Cascading Dropdowns

Select State → District list narrows automatically. Select District → Block narrows. Works identically on web and mobile.

👁️
Conditional Fields

Fields show/hide based on user input. "Jail Training details" only appears when Purpose = "Jail Training." Same rules on web and mobile.

✨
Auto-fill from Links

Select a Volunteer → their name, designation, GPS auto-populate. Reduces errors and speeds up entry.

🛡️
Three Layers of Validation

Simple rules in form definition. Complex rules in Server Scripts (Python). Real-time feedback via Client Scripts. Data is always checked.
